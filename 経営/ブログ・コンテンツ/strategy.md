# ブログ・情報発信戦略 Blog & Content Strategy

更新日：2026-05-02

---

## 発信の目的

1. **信頼の蓄積**：開業前から専門家としての認知を高める
2. **集患への布石**：オンライン睡眠クリニック・将来の実クリニック開業への患者獲得
3. **紹介元への訴求**：他科医師・かかりつけ医へのアピール
4. **患者教育**：正しい医療情報の提供
5. **学習の場**：Web・SEO・マーケティングの実践学習

## 技術スタック（確定）

**Astro + Markdown + GitHub + Cursor + 外部フォーム**

### 選択理由

| 条件 | 根拠 |
|---|---|
| Markdownに慣れている | このリポジトリで毎日書いている |
| Cursorを使っている | AIとのコーディングが得意 |
| 費用を抑えたい | Vercel/Netlifyで無料ホスティング可 |
| 学習の場として使いたい | Astroは学習コスパが高い |
| このリポジトリとの連携 | 非公開メモ → 公開記事への流れを一元化できる |

### ワークフロー

```
このリポジトリ（非公開）          ブログリポジトリ（公開）
経営/ブログ・コンテンツ/   →      src/content/blog/
  article_ideas.md                  記事ネタから執筆・公開
```

## SEO設定（Astro必須設定）

最初から入れておく項目：

| 設定項目 | 内容 |
|---|---|
| OGP / Twitter Card | SNSシェア時の見た目を制御 |
| canonical URL | 重複コンテンツを防ぐ |
| sitemap.xml | 自動生成（@astrojs/sitemap） |
| robots.txt | クローラー制御 |
| JSON-LD（構造化データ） | 医師・医療機関の信頼性向上 |
| meta description | 各記事に必ず設定する |
| `@astrojs/seo` integration | 一括管理 |

## セットアップ手順（学習順）

### Phase 1：環境構築（1〜2週間）

```bash
# Astroプロジェクト作成
npm create astro@latest my-medical-blog

# ブログテンプレートを選択
# GitHubリポジトリを作成してpush
# Vercelと連携してデプロイ（30分で完了）
```

### Phase 2：カスタマイズ（2〜4週間）

- プロフィールページを書く（SEOで最重要）
- SEO設定（sitemap・OGP・JSON-LD）を入れる
- お問い合わせフォームをFormspreeで実装
- プライバシーポリシー・免責事項ページを作る

### Phase 3：記事執筆・公開（継続）

- 月2〜4本のペースで記事を書く
- Cursorで下書き → GitHubにpush → 自動デプロイ
- SNSで記事を宣伝する
